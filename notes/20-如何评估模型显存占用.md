# 如何评估模型显存占用

## 核心公式

**模型参数显存** = 参数量 × 每参数字节数

| 精度 | 每参数字节数 | 7B 模型参数占用 |
|------|------------|----------------|
| FP32 | 4 bytes | ~28 GB |
| FP16 / BF16 | 2 bytes | ~14 GB |
| INT8 | 1 byte | ~7 GB |
| INT4 | 0.5 byte | ~3.5 GB |

## 推理时的显存组成

```
总显存 ≈ 模型权重 + KV Cache + 激活值 + 临时开销
```

1. **模型权重**：`参数量 × 每参数字节数`
2. **KV Cache**：`2 × num_layers × hidden_size × num_kv_heads/num_heads × max_seq_len × batch_size × dtype_bytes`
3. **激活值 / 临时开销**：通常占 5-20% 额外空间

## 快速估算方法

**粗略估算**（推理）：

```
总显存 ≈ 模型参数显存 × 1.2  （低并发、短序列）
总显存 ≈ 模型参数显存 × 1.5~2.0 （高并发、长序列）
```

**实测方法**：

```bash
# 用 vLLM 启动后查看实际占用
python -c "
from vllm import LLM
llm = LLM(model='your-model', gpu_memory_utilization=0.9)
"
# 然后用 nvidia-smi 观察
nvidia-smi
```

## 训练时的显存（供参考）

训练比推理占用大得多：

```
训练显存 ≈ 模型参数 + 梯度 + 优化器状态 + 激活值
```

- Adam 优化器下：FP16 混合精度训练约需 **每参数 18 bytes**
- 7B 模型全参数训练约需 ~120 GB

## 实际例子

以 **Qwen3.5-375B** 为例（FP16）：
- 模型权重：375B × 2 bytes = **~750 GB**
- 需要至少 **10 张 80GB A100** 才能装下权重
- 量化到 INT4：375B × 0.5 = **~188 GB**，约 **3 张 80GB A100**
