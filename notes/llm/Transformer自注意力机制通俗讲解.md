# Transformer 自注意力机制通俗讲解

## 一句话概括

自注意力就是：**句子里的每个词，回头看看其他所有词，决定该重点关注谁。**

---

## 生活类比：课堂讨论

想象你在一个教室里，老师问了一个问题：**"它"指的是谁？**

> "那只猫跳上了桌子，因为**它**很好奇。"

你（代表"它"这个词）需要环顾教室（整个句子），看看哪个同学（哪个词）跟你关系最密切。你发现"猫"跟你关系最大，于是你重点关注了"猫"的信息。

**这就是自注意力在做的事。**

---

## 三个核心角色：Q、K、V

每个词会扮演三个角色，用**找餐厅**来类比：

| 角色 | 类比 | 含义 |
|------|------|------|
| **Q (Query)** | 你的需求："我想吃川菜" | 我在找什么信息？ |
| **K (Key)** | 餐厅招牌："正宗川菜馆" | 我能提供什么类型的信息？ |
| **V (Value)** | 餐厅的实际菜单和菜品 | 我的实际内容是什么？ |

流程：
1. 你拿着 **Q（需求）** 去和每家餐厅的 **K（招牌）** 比较
2. 越匹配的餐厅，你给它的**关注度越高**（权重越大）
3. 最后按关注度加权，获取各餐厅的 **V（菜品信息）**

---

## 手把手走一遍

句子：**"我 喜欢 猫"**

### 第 1 步：每个词生成 Q、K、V

每个词通过三个不同的权重矩阵，变出三个向量：

```
"我"  → Q₁, K₁, V₁
"喜欢" → Q₂, K₂, V₂
"猫"  → Q₃, K₃, V₃
```

### 第 2 步：计算关注度分数

以"喜欢"为例，它拿自己的 Q 去和所有词的 K 做点积（越大越相关）：

```
"喜欢"对"我"的关注度   = Q₂ · K₁ = 3.0
"喜欢"对"喜欢"的关注度 = Q₂ · K₂ = 1.5
"喜欢"对"猫"的关注度   = Q₂ · K₃ = 5.0  ← 最高！
```

### 第 3 步：缩放 + Softmax 归一化

```
除以 √d（防止数值太大）→ [1.5, 0.75, 2.5]
Softmax 转为概率       → [0.26, 0.12, 0.62]
                              ↑          ↑
                           关注较少    重点关注！
```

### 第 4 步：加权求和得到输出

```
"喜欢"的新表示 = 0.26 × V₁ + 0.12 × V₂ + 0.62 × V₃
                  (我)        (喜欢)       (猫)
```

此时"喜欢"这个词的表示里**融入了"猫"的大量信息**，模型就理解了"喜欢"的对象是"猫"。

---

## 为什么要除以 √d？

点积的结果会随着向量维度增大而变大。如果不缩放，Softmax 的输出会趋向于 one-hot（一个接近 1，其余接近 0），梯度几乎为零，训练不动。

```
不缩放: scores = [30, 15, 50] → softmax ≈ [0.00, 0.00, 1.00]  ← 太极端
缩放后: scores = [1.5, 0.75, 2.5] → softmax = [0.26, 0.12, 0.62]  ← 柔和合理
```

---

## 为什么叫"自"注意力？

因为 Q、K、V 都来自**同一个序列**。句子自己看自己，每个词关注同一句话中的其他词。如果 Q 来自一个序列、K/V 来自另一个序列，那就是**交叉注意力**（比如翻译任务中，目标语言关注源语言）。

---

## 一张图总结

```
输入: [我, 喜欢, 猫]
        ↓    ↓    ↓
      生成 Q, K, V
        ↓
      每个词用自己的 Q 和所有词的 K 算相似度
        ↓
      Softmax 得到注意力权重
        ↓
      用权重对所有词的 V 加权求和
        ↓
输出: [我', 喜欢', 猫']   ← 每个词都融合了全局信息
```

核心价值：**不管两个词距离多远，只要相关，注意力就能一步到位地建立联系。** 这就是 Transformer 超越 RNN 的关键所在。
